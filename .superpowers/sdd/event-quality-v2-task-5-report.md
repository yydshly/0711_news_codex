# Event Intelligence v2 Task 5 中文事件网页与处理覆盖报告

日期：2026-07-15

## 实施范围

本任务只实现中文网页与只读查询，没有执行 Event Intelligence v2 回填，也没有修改来源目录、来源 YAML、抓取逻辑或模型决策边界。

复用了现有事件首页、事件列表、事件详情、项目能力页及其导航结构，没有创建第二套站点。事件列表默认只显示当前事件；历史旧版事件通过独立查询入口查看，并在列表和详情明确提示不参与当前首页。

## 查询与展示口径

- 首页窗口固定为最近 24 小时，只展示当前、已确认、AI 相关性不低于 60，且当前版本、六项评分和证据快照完整的事件。真实数量不足时不补低质量事件。
- 事件列表使用事件、当前版本和评分的单次集合投影；缺少当前版本的记录不会进入列表。
- 详情展示 AI 相关性、来源覆盖、来源权威性、时效、互动热度、新颖性六项评分及中文解释，并展示关注理由、限制、证据角色、独立性和独立证据根数量。
- 模型运行摘要只投影模型、用途、结果和延迟四项，不读取或展示提示内容、用量字段和错误原文。
- 处理覆盖窗口固定为最近 72 小时，相关性版本固定为 `relevance-v2`，候选版本固定为 `cluster-v2`。覆盖视图包括窗口条目、已处理、纳入、排除、排除原因、候选、当前事件、旧版事件、模型回退和最近成功完成时间。
- 处理记录通过一次集合查询读取，再在内存中聚合稳定原因码；没有按 RawItem 循环查询。

## 中文页面结果

- 首页首屏增加当前确认事件、当前新兴线索、72 小时处理覆盖、排除数和最后完成时间。
- 事件卡片增加关注理由、中文状态、独立证据数、重要度、可信度、热度和中文增强来源。
- 事件详情增加六项评分卡、综合原因、限制、证据时间线和安全模型运行摘要。
- 项目能力页增加 72 小时事件处理覆盖区块和中文排除原因分布。
- 保留现有受控操作、安全令牌、来源看板和导航入口。

## 安全验证

- 证据链接只允许公开 HTTP(S) 地址，拒绝带用户信息的地址，并在展示前移除查询参数和片段。
- 标题、摘要、关注理由、限制和审计字段均通过受限只读投影；原始正文和完整 payload 不进入网页上下文。
- 模型错误、用量和提示内容没有进入模型运行 DTO。
- 敏感请求头、环境凭据字段和明确的密钥赋值会被隐藏；普通技术语境中的 Prompt 与 token 术语保持可读。
- 模板保持自动转义，并有脚本标签、事件处理属性、查询凭据和敏感审计文本的回归断言。

## 测试与剩余事项

Task 5 聚焦测试覆盖首页门槛、current/legacy 路由、完整快照、六项评分、证据根、模型摘要、处理覆盖集合查询、XSS 和敏感信息。完整项目测试和 Ruff 在提交前重新执行。

本任务没有运行真实 72 小时回填，因此实际窗口覆盖率、真实候选数量、真实当前事件数量和真实模型回退数量由后续 Task 6 验收产生；当前网页会如实显示数据库已有结果，不伪造或补齐数字。
