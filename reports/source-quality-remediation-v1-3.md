# 来源质量修复 v1.3 验收报告

## 结论

本轮对 10 个降级来源完成了协议修复、配置校准和真实网络探测。结果如下：

- 6 个来源连续三轮内容探测成功，样本字段完整率均为 100%；
- 2 个来源接口返回 HTTP 200，但当前没有内容，已明确标记为 `no_content`；
- 2 个 RSS 来源能够返回内容，但 feed 本身缺少摘要字段，已明确标记为 `incomplete_fields`；
- 没有通过降低 90% 完整率门槛、伪造样本或启用 HTML 自动回退获得成功结果；
- 187 个来源 YAML 严格校验通过，重复同步结果为 0 更新，目录同步幂等。

## 修复内容

1. 通用 JSON 与 RSS 探测器现在区分 `no_content` 和 `incomplete_fields`。
2. Bluesky 互动量使用点赞、转发和回复之和。
3. Mastodon 使用专用规范化，支持公开状态与转发正文，字段包括作者、正文、规范链接、发布时间和互动总量。
4. RSS 在 feed 自身提供信息时支持摘要与正文互相回退，但不会访问文章 HTML。
5. 网页将空内容显示为“入口正常，当前没有可用内容”，将字段不足显示为“已获取内容，但缺少必要字段”。
6. 10 个来源定义已改为中文可读说明，并保留 `Fallback:` 机器审计标签。

## 安全与合规检查

- 未使用 Cookie、登录态、验证码处理、代理绕过或非官方抓取；
- 未新增 HTML 自动回退；
- 未改变来源资格晋级规则；`no_content` 和 `incomplete_fields` 都不能满足连续三次成功；
- 报告只记录状态、字段和限制，不包含 API Key、Authorization、Cookie、请求头或完整响应正文；
- MiniMax 不参与来源合规、启用状态或本报告结论。

## 后续建议

- 保持 6 个连续成功来源进入现有稳定性观察，不扩大本轮范围；
- DeepMind 与 Hugging Face 继续作为可用发现入口，但在严格字段口径下保留降级状态；若未来确需正文，应另行设计并审核 HTML/页面读取方案；
- Anthropic Bluesky 和 Qwen3 Releases 按正常轮询间隔等待新内容，不能频繁重试；
- 下一阶段先完成本分支合并前审查和网页验收，再决定是否处理其他来源质量问题。
