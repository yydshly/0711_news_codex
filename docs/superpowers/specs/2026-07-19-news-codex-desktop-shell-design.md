# News Codex Desktop Shell 设计

## 目标

将现有本地 Web/Worker 运行方式包装为用户可感知、可控制的 Windows 桌面应用。登录 Windows 后默认显示日报窗口；用户可隐藏到通知区域、查看服务状态、启动或停止服务，并可明确退出。

## 范围

- 新增 `newsradar desktop` 命令，启动原生窗口与系统托盘。
- 默认窗口嵌入现有 `http://127.0.0.1:<port>/daily-reports`，不复制任何网页业务界面。
- 桌面壳只管理自己启动的 `newsradar serve` 子进程；不会停止既有外部服务、PostgreSQL 或其他 Worker。
- 关闭主窗口隐藏到托盘；托盘提供显示、隐藏、启动服务、停止服务、打开日报、退出。
- 明确“退出”时停止桌面壳拥有的服务子进程并退出托盘。
- 通过用户级 Windows 登录项启动桌面壳，首次默认显示窗口。
- README 说明命令启动、桌面启动、登录启动、隐藏与退出的差别。

## 架构

`DesktopController` 负责端口健康检查、可控子进程、状态快照与退出清理。它通过依赖注入的进程工厂和 HTTP 探针保持可单元测试。

`DesktopApplication` 适配 pywebview 与 pystray：窗口创建后加载日报 URL；关闭事件改为隐藏；托盘动态显示当前服务状态。pywebview 的 GUI 循环在主线程运行，托盘使用 `run_detached`，两者不争用主线程。

Windows 登录项使用当前用户的注册表 `HKCU\\Software\\Microsoft\\Windows\\CurrentVersion\\Run`。写入的命令只指向当前 Python 环境的 `newsradar desktop`，不包含数据库地址或 API 密钥。

## 生命周期

1. `newsradar desktop` 创建控制器并检查本地日报服务。
2. 服务未运行时，控制器以受限环境启动 `newsradar serve --host 127.0.0.1 --port <port>`，轮询有限次数等待 HTTP 200。
3. 创建并显示主窗口；托盘图标显示“运行中”或“需要启动”。
4. 用户关闭窗口时，窗口隐藏且桌面壳继续运行。
5. 用户在托盘选择“停止服务”时，只停止本桌面壳创建的子进程。
6. 用户选择“退出 News Codex”时，停止自有子进程、停止托盘图标并退出 GUI。

## 失败与安全

- 端口被其他服务占用时只显示其可访问状态；绝不按 PID 猜测并杀死进程。
- 服务启动超时后显示中文状态与最近一条安全错误，不无限重试。
- 不读取、展示或写入 `.env` 密钥；子进程沿用当前安全环境。
- 不在每次桌面启动时自动运行数据库迁移。
- 自动日报是否启用继续由现有网页配置决定，桌面启动不改变它。

## 验收

- 控制器可识别已有服务、启动缺失服务、只停止自有服务、超时后给出状态。
- 窗口关闭改为隐藏，托盘菜单能恢复窗口并执行退出清理。
- Windows 登录项命令可创建、读取与移除，且不泄露密钥。
- README 以中文描述四种日常操作路径。
