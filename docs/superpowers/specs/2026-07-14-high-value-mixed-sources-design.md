# News Codex 高价值混合来源波次设计

## 1. 背景与目标

News Codex 已经具备来源目录、协议探测、持久化 Worker、原始条目、事件聚合和中文网页能力，也已经实现 RSS、Reddit、YouTube、Bluesky、Mastodon、Hacker News、GDELT 与 Google News 等抓取器。

当前主要问题不是缺少新的系统架构，而是来源能力没有围绕“真实热点发现”形成清晰、经过真实网络验证的组合：官方来源较多，社区讨论、视频内容、社交热度、专业媒体报道和聚合发现仍未形成稳定闭环。

本波次只完成以下目标：

1. 复用现有抓取器，确认并扩展高价值具体目标。
2. 区分直接抓取、间接发现、等待凭据与失败降级。
3. 对开放来源完成连续三轮真实抓取。
4. 在现有中文网页中展示本波次覆盖、方法、状态和问题。
5. 生成可审计的中文健康报告。

本波次不开发新的摘要、推荐、推送、调度产品或来源抽象层。

## 2. 设计原则

### 2.1 平台与具体目标分离

Provider 表示平台能力，例如 YouTube、Reddit、Techmeme；Source/Target 表示具体频道、账号、社区、查询或 Feed。不得把“整个 YouTube”或“整个 Bluesky”当作已经覆盖的内容来源。

### 2.2 发现与证据分离

- 社区和社交来源承担 `discovery`、`engagement` 和 `context`。
- 聚合来源承担 `discovery`。
- 官方一手与独立专业媒体承担 `evidence`。
- 聚合来源和单条社交内容不得独立确认新闻事实。

### 2.3 能力存在不等于内容已覆盖

网页和报告必须明确区分：

- `direct_ready`：真实获取到内容样本。
- `indirect_ready`：通过聚合入口发现原始媒体内容。
- `blocked`：接口存在，但缺少凭据、审批或付费条件。
- `degraded`：可以访问，但存在超时、结构漂移、内容不足等问题。
- `failed`：本轮真实抓取失败。

只有内容探测或正式抓取成功，才能显示为“已获取内容”。

### 2.4 不增加高风险回退

- Reddit 不回退到 Cookie、登录网页或匿名 `.json` 抓取。
- YouTube 不把全局搜索作为频道日常抓取主路径。
- 不使用登录态、验证码绕过、代理池或非官方高风险爬虫。
- HTML 只作为经过人工审核的来源级备用方式，不作为平台级通用回退。

## 3. 本波次来源范围

### 3.1 Reddit

首批保留并验证三个社区：

- `r/LocalLLaMA`
- `r/MachineLearning`
- `r/artificial`

使用现有 OAuth 抓取器。缺少凭据时运行能力探测，页面显示申请条件和当前阻塞，不执行内容抓取。启用前补齐删除内容与作者删除状态的处理验证。

### 3.2 YouTube

首批形成约八个具体频道：

- 五个经过官方主页交叉确认的 AI 公司、研究机构或开发者官方频道。
- 三个经过人工确认的行业媒体、研究者、访谈或专家观点频道。

主路径：

1. 使用 `channels.list` 确认 Channel ID 和 uploads playlist。
2. 使用 `playlistItems.list` 获取最新上传视频。
3. 必要时使用 `videos.list` 补齐浏览、点赞和评论指标。
4. 频道 Atom Feed 作为无凭据备用方式。
5. 字幕获取只作为可选增强，字幕失败不影响视频条目落库。

现有 `openai-youtube` 从 `degraded` 重新验证。目标目录不得只保存展示名称，必须保存稳定 Channel ID、官方身份链接和备用 Feed。

### 3.3 Bluesky

首批形成约六个具体账号目标。优先选择：

- 使用机构域名作为 handle 的账号。
- 能从机构或个人官方主页反向确认的账号。
- 持续发布 AI 产品、研究或开发动态的账号。

使用公共 AppView 的作者 Feed；搜索接口只作为补充发现，不代替账号身份审核。保存作者 DID、handle、原帖 URI、回复/转发/点赞指标和线程根。

### 3.4 Mastodon

首批形成约四个具体目标，可包含账号与受控主题时间线：

- 账号目标优先于公共全站时间线。
- 主题时间线必须绑定具体实例和标签。
- 公共时间线只允许 `local=true` 且必须在 YAML 中明确登记。

每个实例单独记录匿名访问能力、限流和失败状态，不把一个实例的结论推广到整个 Mastodon 网络。

### 3.5 Hacker News

复用现有三个目标：

- Top Stories
- New Stories
- Best Stories

继续保存分数、评论数、作者、原文地址和 HN 讨论地址。内容进入候选事件前由规则进行 AI/技术相关性过滤，不新增第三方搜索依赖。

### 3.6 Techmeme、GDELT 与 Google News

Techmeme 使用官方 Feed，作为专业媒体集中报道和突发科技新闻的发现信号。

GDELT 只作为广域发现层：

- 缩小查询条件。
- 限制返回数量。
- 使用明确的连接与读取超时。
- 遇到 429、5xx 和超时按现有 Worker 重试策略处理。
- 必须解析并保存原始发布者和原文地址；无法确认原始归属的条目隔离。

Google News 使用 RSS，建立五类主题查询：

1. 模型与产品发布。
2. AI 研究突破。
3. 芯片与算力。
4. 公司、融资与并购。
5. 政策、安全与版权。

同时为缺少稳定公开 RSS/API 的重要媒体建立发布者限定查询。Google News 地址属于发现地址，规范化后必须保存最终原文地址与原始媒体。

### 3.7 专业媒体

第一层验证现有直接入口：

- BBC Technology
- Ars Technica
- CNBC Technology
- TechCrunch
- The Verge
- WIRED
- The Guardian Technology
- MIT Technology Review
- VentureBeat

第二层为 Reuters、AP、Bloomberg、Financial Times、WSJ 等受限媒体建立或验证 Google News 间接发现入口。没有稳定官方接口时保持 `indirect_ready`，不得显示为直接抓取，也不得自动启用 HTML。

## 4. 数据流

```text
已审核 YAML 目标
  → 内容/能力探测
  → 持久化 fetch operation
  → Worker 调用现有协议抓取器
  → RawItem 规范化与来源归属
  → 来源波次状态聚合
  → 中文网页与健康报告
```

本波次不改变 RawItem 主模型。需要补充的波次信息优先由查询层从 Source、FetchRun、Operation 和 RawItem 聚合，避免引入重复状态表。只有现有数据无法表达目标验证结果时，才增加最小数据库迁移。

## 5. 失败与可靠性

- 单个目标失败不终止同批其他目标。
- 所有网络调用必须使用现有超时、并发限制和 Worker 心跳。
- 401/403 映射为凭据或审批阻塞，不进行匿名网页回退。
- 429 记录重试时间并进入可重试状态。
- 5xx、连接失败和读取超时按有限次数重试，最终显示可读中文原因。
- Schema 漂移不得静默产生空成功；字段不足时显示 `degraded`。
- 日志继续执行敏感字段清理，不记录 Token、API Key 或 OAuth Secret。

## 6. 中文网页展示

在现有来源感知台中增加“高价值混合来源波次”入口和视图，不创建第二套网站。

页面展示：

- 本波次平台数、具体目标数。
- 直接抓取、间接发现、等待凭据、降级和失败数量。
- 按平台查看每个目标的身份、角色、方法和备用方法。
- 最近三次探测或抓取时间、结果、条目数和最新内容时间。
- 失败原因、是否影响整个平台、下一步人工操作。
- 链接到来源详情、抓取批次、原始条目和原文。

中文描述必须使用产品语言，例如“等待 Reddit 开发者凭据”，避免只展示 `blocked`、`degraded` 等内部枚举。

## 7. 实施边界

优先复用以下现有模块：

- `newsradar.ingestion.fetchers`
- `newsradar.operations.worker`
- `newsradar.sources` 与 `newsradar.providers`
- `newsradar.web` 现有查询层和模板

允许的代码改动：

- 修复现有抓取器的真实兼容性问题。
- 增加经过审核的具体目标 YAML。
- 增加波次查询、中文视图和报告。
- 增加必要的单元、集成和真实网络验收脚本。

禁止的扩张：

- 新建另一套抓取框架。
- 重做 Provider/Target/RawItem 架构。
- 开发摘要、推荐、推送或常驻调度产品。
- 自动生成未经人工身份确认的账号目标。

## 8. 验收标准

1. Reddit、YouTube、Bluesky、Mastodon、HN、Techmeme、GDELT、Google News 和专业媒体九类全部有明确结果。
2. 所有开放且可用的直接入口完成连续三轮真实抓取。
3. Reddit 在无凭据时正确显示能力阻塞；有凭据时完成真实内容验证。
4. YouTube 同时覆盖官方机构和专家观点频道，主路径不依赖全局搜索。
5. 聚合条目保存发现地址、原始媒体和最终原文地址；无法确认归属时隔离。
6. 页面能从波次概览下钻到来源、抓取记录和原始条目。
7. 单个目标失败不会阻塞整批；超时、429、字段漂移均有中文可读结论。
8. 生成中文报告，逐项列出性质、方法、备用方式、样本、风险和结论。
9. 全量测试和 Ruff 通过，真实网络验收不依赖 MiniMax。

## 9. 模型使用

本波次来源合规、身份确认和启用决策由规则及人工证据完成，不交给 MiniMax。

MiniMax 只允许用于已有边界内的样本主题分类和错误解释；模型不可用不得影响抓取、探测、网页状态或报告生成。
