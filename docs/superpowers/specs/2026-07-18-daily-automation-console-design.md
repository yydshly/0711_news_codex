# News Codex 日报自动化控制台与存档保留设计

日期：2026-07-18  
状态：待用户书面复核  
目标版本：日报生产化第二阶段

## 1. 目标

在现有自动日报、持久队列、取消、恢复、中文双版本报告和双版本音频能力上，增加完全由网页管理的每日自动执行能力，并为日报存档增加置顶、回收站、恢复和低负载永久清理。

用户不需要操作命令行或 Windows 计划任务。网页负责配置和展示；现有 Worker 负责低成本调度与任务执行。新功能默认暂停，只有用户在网页明确启用后才会自动运行。

## 2. 已确认的产品规则

- 每天北京时间 07:30 自动执行，包括周末。
- 自动执行固定读取最近 24 小时；网页“立即执行”继续允许选择 24、48 或 72 小时。
- 暂停只阻止后续计划，当前正在运行的日报继续完成。
- 取消当前任务使用现有取消机制，与暂停计划分开。
- 电脑关机或 Worker 停止而错过 07:30 时，恢复后只补跑当天一次，不补齐更早日期。
- 同日、同窗口、同波次重复触发时复用已有任务；失败或取消后允许重试。
- 默认资源档位为“标准”：全局抓取并发 8、单平台并发 2、模型并发 2。
- 日报默认保留最近 90 天；置顶日报不自动清理。
- 超过 90 天的非置顶日报自动移入回收站。
- 回收站保留 30 天，期间可恢复；到期后低优先级永久清理。
- 永久清理只删除日报、日报审核和日报音频，不删除 RawItem、事件、来源、证据或抓取历史。

## 3. 非目标

- 不重新实现来源抓取器、事件管线、中文增强或音频生成。
- 不增加推荐、推送、邮件、聊天工具通知或复杂摘要。
- 不让网页进程直接执行抓取、模型调用或文件清理。
- 不支持中断网络请求后从请求中间位置恢复；运行中的主任务只支持现有安全取消。
- 第一版不开放高于现有安全并发的“快速”资源档位。
- 不在测试或部署过程中自动删除现有真实日报。

## 4. 总体架构

### 4.1 组件职责

1. **日报自动化配置**
   - 保存启用状态、执行时间、时区、自动窗口、资源档位和最近调度结果。
   - 作为网页和 Worker 的唯一配置来源。

2. **Worker 调度检查器**
   - 集成到现有 Worker 空闲循环，不创建忙轮询线程。
   - 每 60 秒最多检查一次到期配置。
   - 到期后只提交现有自动日报任务，不直接运行抓取。

3. **现有自动日报管线**
   - 继续负责高价值波次、事件处理、中文审核、归档和双版本音频。
   - 继续使用持久队列、重试、取消、心跳和恢复。

4. **网页自动化控制台**
   - 修改配置、显示调度状态、立即执行、暂停计划、取消当前任务。
   - 只读取配置和现有运行摘要，不扫描全量 RawItem。

5. **日报保留与清理器**
   - 低频扫描超过保留期的日报并移入回收站。
   - 小批量永久清理到期日报与音频文件。

### 4.2 调度数据流

```text
Worker 空闲循环
  → 每 60 秒检查一次自动化配置
  → 判断是否到期或今天需要补跑
  → 获取数据库调度锁
  → 再次检查当天任务与配置状态
  → 提交现有 DailyAutopilotRun
  → 持久队列推进全部阶段
  → 更新上次运行与下一次运行时间
  → 网页读取配置和运行摘要
```

网页服务不持有定时器。即使网页服务重启，也不会影响 Worker 的调度判断。

## 5. 持久化设计

### 5.1 自动化配置

新增单例配置记录 `daily_automation_config`，只允许一条当前配置：

- `id`：固定主键。
- `enabled`：默认 `false`。
- `timezone`：固定 `Asia/Shanghai`。
- `daily_time`：默认 `07:30`。
- `window_hours`：自动计划固定为 `24`。
- `resource_profile`：第一版为 `standard`，保留 `power_saver` 枚举位置。
- `last_scheduled_date`：最近成功提交计划的北京时间日期。
- `last_run_id`：最近关联的自动日报任务，可为空。
- `next_run_at`：下一次 UTC 执行时间，建立索引。
- `created_at`、`updated_at`：审计时间。

所有网页修改在事务中重新计算 `next_run_at`。暂停时保留配置但不提交新任务；重新启用时根据北京时间计算下一次执行，并判断当天是否需要补跑。

### 5.2 日报保留字段

在 `daily_reports` 增加：

- `pinned_at`：非空表示置顶保护。
- `deleted_at`：非空表示位于回收站。
- `purge_after`：默认等于 `deleted_at + 30 天`。

普通日报查询默认增加 `deleted_at IS NULL`。回收站查询只读取 `deleted_at IS NOT NULL`，按 `purge_after` 升序分页。置顶只阻止自动移入回收站，不阻止用户明确手动删除。

### 5.3 永久清理关系

永久清理前执行以下顺序：

1. 校验日报仍在回收站且已经到期，或请求来自用户明确永久删除。
2. 校验没有进行中的日报或音频操作。
3. 收集该日报当前音频文件的受控本地路径。
4. 删除音频文件；文件不存在视为幂等成功。
5. 在数据库事务中将历史 `DailyAutopilotRun.daily_report_id` 置空，并在运行摘要中保留“原日报已清理”的安全标记。
6. 删除 `DailyReportRecord`，依靠现有受控级联清理日报条目、审核和音频记录。
7. 保留关联的 Operation、FetchRun、RawItem、Event、Source 和 Evidence 数据。

如果文件删除失败，不删除数据库日报；记录中文诊断并进行有限重试。如果文件已删除而数据库事务失败，重试时将“文件不存在”视为成功，然后继续数据库清理。

## 6. 调度与并发语义

- 调度判断必须在数据库锁内二次确认，防止两个 Worker 重复提交。
- PostgreSQL 使用事务级 advisory lock；测试数据库使用同一事务边界验证幂等语义。
- 当天存在 `queued`、`running` 或 `succeeded` 的相同任务时复用。
- 当天只有 `failed` 或 `cancelled` 任务时允许创建重试任务。
- 补跑只判断当前北京时间日期，不循环创建历史日报。
- 页面“立即执行”与定时器同时触发时，调用同一个命令服务和同一幂等规则。
- 暂停配置不会修改当前运行任务；取消按钮调用现有自动日报取消路由。
- 单个来源失败继续允许主任务以部分成功推进。
- 报告成功但单份音频失败时，报告保持可读并显示“音频部分失败”；只重试失败音频，不重新抓取或重建日报。

## 7. 网页设计

### 7.1 自动日报控制台

在 `/daily-reports` 顶部增加控制台，展示：

- 状态：已启用或已暂停。
- 计划：每天北京时间 07:30，最近 24 小时。
- 资源档位：标准。
- 上次运行时间、状态、耗时和任务链接。
- 下次计划执行时间。
- Worker 在线状态；离线或过期时显示中文诊断。
- 当前任务阶段和覆盖漏斗。

提供以下受控操作：

- 启用自动日报。
- 暂停后续计划。
- 立即执行，可选 24、48 或 72 小时。
- 取消当前任务。
- 打开当前任务详情。

运行中的覆盖漏斗保持现有口径：计划目标、成功抓取、受阻目标、形成事件、进入决策简报、进入情报全览和双版本音频状态。

### 7.2 日报存档

在现有存档表增加：

- 单条选择和当前页全选。
- 置顶保护与取消置顶。
- 移入回收站。
- 批量移入回收站。
- 筛选：全部、最近 7 天、最近 30 天、已置顶。
- 自动保留剩余天数。

正在生成、仍有音频任务运行或正在清理的日报禁用删除，并显示具体中文原因。批量操作逐项返回结果，单条失败不阻塞其他日报。

### 7.3 回收站

新增 `/daily-reports/trash`：

- 显示删除时间和预计永久清理时间。
- 支持单条恢复、批量恢复。
- 支持用户二次确认后立即永久删除。
- 默认分页，不一次加载全部历史。
- 直接访问回收站日报详情时，不渲染普通正文，显示回收站状态、恢复入口和清理时间。

所有写操作使用现有 POST、安全 action token、同源校验和中文错误映射。

## 8. 网页接口边界

第一版使用以下固定接口路径，并沿用现有 FastAPI 路由、安全校验和中文错误映射：

- `POST /daily-automation/enable`
- `POST /daily-automation/pause`
- `POST /daily-automation/run-now`
- `POST /daily-reports/{id}/pin`
- `POST /daily-reports/{id}/unpin`
- `POST /daily-reports/{id}/trash`
- `POST /daily-reports/{id}/restore`
- `POST /daily-reports/{id}/purge`
- `POST /daily-reports/bulk/trash`
- `POST /daily-reports/bulk/restore`
- `GET /daily-reports/trash`

批量请求限制当前页最多 50 条，防止一次请求创建过大的事务或清理任务。

## 9. 性能约束

- Worker 每 60 秒最多执行一次轻量配置查询；空闲时使用现有休眠机制，不增加忙循环。
- `next_run_at`、`deleted_at`、`purge_after` 和 `pinned_at` 的查询路径具有适当索引。
- 标准档位固定为全局抓取并发 8、单平台并发 2、模型并发 2。
- 同时最多存在一个自动日报主任务。
- 网页状态查询只读取自动化配置、当前任务和持久化摘要。
- 有活动任务时页面最多每 10 秒刷新一次轻量状态；无活动任务时停止自动刷新。
- 保留扫描每天最多执行一次，使用日期和索引筛选。
- 永久清理每批最多 20 份日报；单批结束后主动让出 Worker，不长期占用 CPU 或磁盘。
- 空闲 Worker 连续 5 分钟不得出现忙循环或持续占用一个 CPU 核心。

## 10. 中文诊断与恢复

网页必须区分并给出下一步：

- 自动计划已暂停。
- Worker 离线。
- 今日计划尚未到期。
- 今日任务正在运行。
- 今日任务已复用。
- 错过时间后已补跑。
- 部分来源受阻但日报继续。
- 报告完成、音频部分失败。
- MiniMax 文本回退。
- 日报因活动任务不能删除。
- 音频文件清理失败。
- 永久清理失败并等待重试。

连续失败不得形成无上限快速重试。网络和文件操作继续使用现有超时、有限重试、取消检查点和结构化日志。

## 11. 测试设计

### 11.1 调度单元测试

- 北京时间 07:30 到期判断。
- 跨日期和 UTC 转换。
- 暂停与重新启用。
- 当日漏跑补跑一次。
- 不补齐更早日期。
- 同日幂等和失败后重试。
- 两个 Worker 争抢时只创建一个任务。

### 11.2 Worker 与集成测试

- 空闲检查使用受控时钟，不出现忙循环。
- 调度器只提交任务，不直接调用网络。
- 报告成功但音频失败时只重试音频。
- Worker 重启后从持久状态恢复。
- 清理任务每批不超过 20 份并可恢复。

### 11.3 网页测试

- 控制台正确显示启用、暂停、离线、到期和运行状态。
- 所有写操作要求 action token 和同源请求。
- 立即执行与定时执行复用同一任务。
- 存档筛选、置顶、移入回收站、恢复和批量操作。
- 活动日报删除被拒绝并显示中文原因。
- 回收站日报不泄漏普通正文。

### 11.4 永久清理测试

- 级联删除日报条目、决策审核、全览审核和音频记录。
- 保留 RawItem、事件、来源、证据和抓取历史。
- 自动日报历史解除报告链接但保持可查看。
- 文件不存在时幂等成功。
- 文件删除失败时数据库日报仍存在。
- 数据库事务失败后可以安全重试。

### 11.5 完整验收

- 完整 `pytest`、`ruff` 和差异检查通过。
- 使用测试日报验收控制台、回收站和恢复，不删除现有真实日报。
- 使用真实网页确认页面换行、状态、中文诊断和移动端基本可读性。
- 观察空闲 Worker 5 分钟，确认调度检查频率和 CPU 没有异常增长。
- 部署后自动化默认暂停；用户在网页明确启用后，页面显示下一次执行时间。
- 第一次真实自动执行安排在下一个自然日北京时间 07:30。

## 12. 分阶段交付

### 里程碑 A：持久化调度与网页控制台

- 自动化配置、数据库调度锁、Worker 到期检查。
- 启用、暂停、立即执行、状态展示和补跑。
- 复用现有自动日报与取消能力。

### 里程碑 B：日报保留、置顶与回收站

- 90 天保留、置顶保护、手动与批量移入回收站。
- 回收站列表、恢复和访问隔离。

### 里程碑 C：低负载永久清理与生产验收

- 30 天到期清理、音频文件处理、运行历史解链。
- 小批次恢复、中文诊断、性能观察和真实网页验收。

每个里程碑完成后统一汇报。未经确认不合并、不推送；不读取或提交 `.env`，不触碰用户保留的未提交报告。
