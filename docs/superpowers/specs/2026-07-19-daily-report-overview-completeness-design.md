# 日报全览完整性设计

## 目标

自动日报不得因展示字段异常而静默丢失事件。决策简报保持精简；情报全览和完整报告保留本次事件快照中的全部可见事件，并对字段不完整的事件进行中文降级说明。

## 行为

- 决策简报仅收录可构造完整展示快照的确认、热点或信号事件。
- 情报全览收录全部确认、热点或信号事件。完整条目构造失败时，生成“待补齐展示数据”条目，而非跳过。
- 降级条目保留事件编号、版本、已有标题、发布时间、证据链接及中文原因；不可用字段显示为空缺，不伪造事实。
- 生成摘要记录总降级数和按原因聚合的计数；日报页面展示该信息。
- 自动日报详情和日报列表提供明确的最新任务/最新日报入口；任务进行中每十秒刷新，完成后显示打开日报链接。
- 不重新抓取来源；修订日报可复用既有固定事件快照。

## 原因编码

- `event_detail_unavailable`：事件详情在固定操作快照中无法读取。
- `invalid_snapshot_datetime`：时间字段异常。
- `invalid_snapshot_evidence`：证据结构异常。
- `invalid_snapshot_limitations`：限制说明结构异常。

## 验收

1. 一次含 11 个可见事件、其中 7 个完整快照构造失败的运行，情报全览仍有 11 条，决策简报只保留完整条目。
2. 降级条目和日报摘要显示可理解的中文原因。
3. 既有安全 URL 清洗继续生效。
4. 自动日报页面可导航到最新任务或生成的日报。
