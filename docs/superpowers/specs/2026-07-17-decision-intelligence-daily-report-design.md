# 决策与情报全览日报设计

## 目标

将现有的事件快照、证据评估、人工中文审核、来源运行状态和持久 Worker 编排为一份可阅读、可审计、可播放的中文日报。它服务个人决策，不重新抓取来源、不改变事件事实，也不让 MiniMax 决定来源是否合法或事件是否已确认。

## 阅读层级

同一份日报固定为两个层级。

1. **决策简报**只读取已纳入条目的固定快照及其最新人工审核。它给出当日结论、按确认状态区分的重要事项、待补证风险和可执行的审核建议。它是默认展开内容，也是语音稿来源。
2. **情报全览**从该日报绑定的完成事件操作快照读取全部可展示事件（confirmed、hotspot、signal），按类别和确认状态分组。它显示中文标题、概述、影响、证据根数、限制、原文链接和是否已收录进决策简报；它不把 RawItem 当作独立新闻重复展示。
3. **来源运行状态**只汇总已存在的来源、Target、最近抓取批次和可定位的阻塞原因；它不触发任何探测或抓取。

## 语音

归档日报可创建两个独立的 MiniMax 音频成品：`decision` 和 `overview`。第一版自动/手动入口只生成决策稿；全览稿由用户按需创建。两者都使用 `speech-2.8-hd`、MP3、32 kHz、128 kbps、单声道、`male-qn-qingse`、语速 1.0。

语音稿由固定日报快照和审核字段确定性拼接，不调用文本模型改写事实。MiniMax 仅把脚本转换成音频。语音请求通过 Worker 持久队列执行，具有超时、有限重试、心跳、取消、结构化日志和脱敏中文诊断。API 响应的十六进制音频在服务端写入受控本地目录，数据库仅记录相对路径和校验和；密钥绝不写入日志、数据库、网页或音频元数据。

## 数据与一致性

新增 `daily_report_audio_artifacts` 表保存每次生成尝试：日报、成品类型、脚本及哈希、模型/音色/音频参数、状态、字符数、MiniMax trace ID、时长、相对文件路径、校验和、错误代码/中文信息及生成时间。一个报告/类型可有多次历史尝试，页面只播放最新成功成品。

新增 `daily_report_audio` 持久操作类型。入队请求携带 `daily_report_id`、`rendition` 和脚本哈希；Worker 在生成前重新核对归档状态和哈希，防止草稿或过期内容生成音频。缺密钥、鉴权/权限、额度、响应格式、网络超时及写盘失败分别映射为稳定错误码。只有临时网络、429、5xx 才允许有限重试。

配置使用独立的 `MINIMAX_TTS_API_KEY`（`SecretStr`），不复用或暴露文字模型配置。缺失配置只会让音频任务失败并显示中文原因，绝不影响日报文字阅读。

## 页面与交互

日报详情页顶部增加“决策简报”阅读卡和音频播放器。归档页面允许创建决策音频、按需创建全览音频；草稿不可生成，以保证音频与已定稿内容一致。播放器提供播放/暂停、下载链接、模型/音色/字符数/生成时间，以及失败后的中文原因和安全重试按钮。

情报全览默认折叠，按类别展示该完成事件快照中的所有可展示事件。每条都链接到现有事件详情和公开原始证据。来源运行状态使用只读查询投影，显示成功、可修复、需用户操作和近期不处理的数量与中文原因。

## 验收

- 决策稿不含排除或重复项，并明确标出 `needs_evidence`。
- 全览可从固定操作快照重建，不依赖当前事件状态。
- Worker 对模拟的 MiniMax 成功、429、鉴权失败、坏音频、取消和断电恢复均有测试。
- 密钥不会出现在 HTML、日志、数据库字段或断言输出中。
- 实际网页能显示归档日报的决策/全览层级和生成/播放状态；若本机已配置 Token Plan Key，可进行一次人工确认后的真实 `speech-2.8-hd` 生成验收。
