# 自动日报完整中文增强设计

## 目标

让自动日报中最终进入决策简报或情报全览的每个唯一事件都获得可阅读的简体中文标题与中文文章概述，并继续生成中文审核建议、中文证据评价和两份中文语音。模型调用失败只能影响对应条目的中文表达，不能阻塞其他条目、日报归档或音频生成。

本设计只修正自动日报的中文内容增强与诊断，不扩充来源、不重新探测 187 个目标、不改变事件聚类、来源启用、证据根、确认状态或收录规则。

## 已确认的根因

2026-07-18 的真实自动日报任务 `#3` 完成了 41/41 个目标处理、34 个成功抓取、21 个事件、16 个日报条目和两份音频，但只有 1 条获得中文模型增强，其余 15 条沿用了英文标题。

这不是 MiniMax 请求连续失败。数据库在该波次只记录了 1 次 `event_enrichment` 成功调用。现有事件管线只对 `hotspot` 或排名分数至少 60 的 `signal` 调用模型；其他能够进入日报的 `signal` 不调用模型，直接使用事件规则回退。日报页面随后把所有非模型事件统一解释为“中文模型不可用”，把“未调用”误写成了“调用失败”。

事件管线的调用门槛是全局事件成本控制策略，不应为了日报而放宽，否则所有事件构建都会增加模型调用，并违背低价值线索不自动调用模型的现有边界。

## 方案选择

采用独立的“自动日报中文增强”阶段，位于日报草稿生成之后、审核版本写入之前。该阶段只处理已经固定在这份日报中的唯一事件，并把同一份中文结果同时用于决策简报和情报全览。

不采用以下方案：

- 不把事件管线改成对全部候选调用 MiniMax，因为这会影响非日报任务、增加全局成本，并重复处理最终不会进入日报的事件。
- 不在网页请求或手动日报生成时调用 MiniMax，因为网页必须立即返回，手动日报也应继续保持只读、可复现。
- 不用简单词典翻译代替模型，因为它无法可靠生成文章概述，容易造成事实变形。

## 完整流程

自动日报链路调整为：

```text
真实高价值内容波次
→ 固定事件清单
→ 生成日报草稿
→ 自动日报中文增强
→ 写入决策版与全览版审核版本
→ 原子归档并提交两份音频
→ 等待音频并完成
```

现有 `write_reviews` 阶段保留其持久化职责，但内部改为先为日报中的唯一事件生成中文增强结果，再用确定性规则生成两类审核记录。旧的自动日报运行记录和已经归档的日报不回写、不删除，也不会被新逻辑自动重算。

## 条目范围与成本边界

处理范围是以下集合的并集，按优先级去重：

1. 决策简报中的 confirmed 条目；
2. 决策简报中的 emerging 条目；
3. 情报全览中尚未出现的其他条目。

同一 `event_id + event_version_number` 在一份日报中最多调用一次模型，决策版和全览版共用结果。决策简报仍保持 confirmed 最多 20 条、emerging 最多 20 条；情报全览仍保存固定快照中的全部 confirmed、hotspot 和 signal，不因模型额度删除内容。

新增本地安全配置 `daily_report_model_max_items`，默认 60。正常高价值日报应全部落在该上限内。超过上限时，剩余条目仍完整进入全览，只使用规则回退，并明确记录 `budget_limit`；不得静默丢弃内容或把它描述成模型故障。

每条请求使用现有快速模型、现有 45 秒总超时和最多一次结构修复重试；并发不超过 2。调用前和每条完成后检查 Worker 取消与租约。按照默认上限，最坏网络等待仍受自动日报 Operation 的 30 分钟总超时约束。

## 模型职责与输入边界

MiniMax 只返回：

- `zh_title`：简洁的简体中文标题；
- `zh_summary`：基于提供材料的简体中文文章概述。

模型不得返回或改变：

- 来源是否合法、是否启用、是否需要凭据或审批；
- 事件是否 confirmed；
- 独立证据根数量；
- `keep`、`needs_evidence`、`exclude` 或 `duplicate` 决策；
- 原始 URL、发布时间和重复关系。

输入只使用日报固定快照中已经存在的标题、摘要、发布者、证据角色、证据限制和确认说明。URL 不进入提示词；所有互联网文本都带不可信内容前言，并按字段截断。模型输出必须通过严格结构校验、现有问号乱码检查、长度检查和“至少包含一个中日韩统一表意文字”的中文检查。未通过即视为该条目回退，不能把英文原文标记为中文增强成功。

## 审核规则保持确定性

模型成功时，自动审核使用模型生成的 `zh_title` 和 `zh_summary`；模型回退时，使用事件快照中的现有标题和概述。随后继续由现有规则生成：

- confirmed 或至少两个独立证据根：`keep`；
- 其他可展示事件：`needs_evidence`；
- 中文审核建议和中文证据评价使用确定性模板；
- 人工后续仍可修订为 `exclude` 或 `duplicate`。

因此，模型不可用不会把待补证信号升级为确认，也不会改变日报收录范围。音频只读取已经保存并通过完整性校验的审核版本。

## 持久化、恢复与幂等

不新增数据库表或列，不需要数据库迁移。

现有审核版本表继续保存最终中文标题、概述、建议和证据评价。`daily_reports.generation_summary` 增加 `daily_chinese_enrichment` 安全摘要：

- 唯一候选数、模型额度、已处理数；
- 模型成功数、规则回退数、额度回退数；
- 使用的模型名称；
- 按安全错误码聚合的失败数量；
- 每个 `event_id + event_version_number` 的 `model`、`rule_fallback` 或 `budget_limit` 来源；
- 与该日报相关的 `model_usage` 记录 ID。

模型正文、提示词、凭据和接口响应不得写入诊断摘要。每次模型尝试仍写入现有 `model_usage` 表，只保存模型名、token、延迟、结果和安全错误码。

每个唯一事件以一个短事务写入两类审核记录和对应诊断状态。重试 `write_reviews` 时：

- 如果该事件已有最终诊断状态且所需审核记录完整，直接跳过；
- 如果模型调用完成但事务尚未提交即进程退出，允许再次调用一次，符合现有至少一次执行语义；
- 不得为已经完整处理的事件重复调用模型或创建无意义的新审核修订；
- 某条模型调用异常只生成该条规则回退，循环继续处理下一条。

## 失败分类与中文诊断

页面和结构化日志必须区分：

- `model`：MiniMax 调用成功且中文输出有效；
- `no_api_key`：未配置文本模型凭据，未发起请求；
- `timeout`：达到单条总超时；
- `http_401` / `http_403`：凭据无效或缺少文本模型权限；
- `http_429`：达到平台频率或额度限制；
- `http_5xx` / `transport_error`：平台或网络异常；
- `schema_validation_failed` / `non_chinese_output`：返回内容不能安全使用；
- `budget_limit`：超过本期本地安全条目上限，未发起请求；
- `rule_only`：明确不需要模型的兼容或人工路径。

不得再把所有规则回退统一显示成“中文模型不可用”。自动日报可以在部分或全部条目回退时成功完成，但必须显示真实成功数、回退数、主要原因和下一步动作。

## 网页展示

自动日报任务页增加“中文增强”指标：候选、MiniMax 成功、规则回退、额度回退和主要错误码。日报详情页的全局提示优先读取新的增强摘要，并显示“16 条中 15 条由 MiniMax 生成、1 条规则回退”这类准确结论。

每个决策版和全览版条目显示小型来源标签：

- `中文增强：MiniMax`；
- `中文增强：规则回退（超时）`；
- `中文增强：规则回退（未配置）`；
- `中文增强：规则回退（安全上限）`。

标签只解释文案来源，不暗示事实已经确认。原文链接、证据限制、确认说明、审核历史和两份播放器保持现有布局。

## 测试驱动与验收

实现前先增加失败测试，至少覆盖：

- 一份日报中同一事件在决策版和全览版只调用一次模型；
- 所有正常返回条目获得简体中文标题和概述；
- 英文或问号损坏输出不能被记录为模型成功；
- 单条超时、429、结构错误或异常只回退该条，后续条目继续；
- 未配置 Key 时不发起网络请求，报告仍能归档并生成两份音频；
- 模型不能改变确认状态、证据根和审核决策；
- 超过 60 条时内容不丢失，超额条目标记 `budget_limit`；
- Worker 续跑跳过已完整处理事件，不重复模型调用或审核修订；
- 手动日报生成和人工修订不调用 MiniMax；
- 页面准确区分未调用、调用成功和调用失败；
- 现有问号乱码防线、双音频原子提交和精确 Operation 快照继续通过。

真实网页验收必须新建一轮 24 小时自动日报，不重试旧报告。验收结果至少包括：真实内容波次执行完成；日报所有模型成功条目均显示中文标题和中文概述；任何回退条目都有逐项中文原因；决策版和全览版内容、原文链接与两份音频均可用。最后运行完整 `pytest`、`ruff check .` 和浏览器验收。

## 非目标

- 不重新探测或扩充来源；
- 不修复 Reddit、Anthropic、GDELT、AP 或 Reuters 的外部访问状态；
- 不改变事件确认、证据和聚类逻辑；
- 不为手动日报自动产生模型费用；
- 不实现定时日报、推送或推荐；
- 不重写或删除已有日报和失败运行。
