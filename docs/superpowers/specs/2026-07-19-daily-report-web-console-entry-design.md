# 中文日报本地网页控制台入口设计

## 目标

让桌面应用用户在中文日报页一眼知道本地网页入口，并能从该页打开默认浏览器或复制地址。

## 范围

在中文日报页面顶部增加一张只读的“本地网页控制台”卡片：

- 显示当前页面的本地控制台地址 `http://127.0.0.1:8767/daily-reports`；
- 说明该地址仅在本机可访问；
- 提供普通链接形式的“在默认浏览器打开”；
- 提供“复制地址”按钮，浏览器不支持剪贴板时显示中文失败提示。

地址从当前请求的 URL 生成，只允许 `http`、`https` 协议，并保留当前主机、端口和路径。它不读取环境变量、不显示凭据，也不把本机服务暴露到局域网或公网。

## 不做的事

- 不改变桌面服务、Worker、自动日报或音频任务；
- 不新增公网部署、身份认证或分享链接；
- 不新增数据库字段或迁移；
- 不新增外部网络请求。

## 实现边界

Web 路由负责构造安全的本机控制台 URL 并传给模板；模板只展示该值。复制行为放在现有前端脚本中，通过页面数据属性获得地址，避免内联脚本和硬编码端口。

## 验收与测试

1. `GET /daily-reports` 渲染本机控制台标题、说明、浏览器链接和复制按钮。
2. 页面链接使用请求的本机地址与 `/daily-reports` 路径。
3. 复制按钮具备可访问名称和状态反馈容器。
4. 现有日报控制台、自动化表单和存档功能不变。
5. 运行日报页面测试、前端相关测试、Ruff 和真实本地页面验收。
