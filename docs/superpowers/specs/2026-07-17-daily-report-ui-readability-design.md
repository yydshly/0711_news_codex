# 日报详情页可读性改造设计

## 目标

让用户进入日报详情页后立即看清“本期共有多少条、决策版收录多少条、多少条待补证、多少条未收录”，并能逐条阅读中文概述、审核建议和证据评价。

## 已确认口径

- 日报生成上限保持不变：确认与线索栏目各最多 20 条。
- “决策版条目”来自日报中 `included=True` 的条目，不设置 3 条硬上限。
- “情报全览”来自日报绑定的事件运行快照，展示其中全部确认、热点和信号事件。
- 待补证条目可以同时属于决策收录；未收录包括排除和重复。
- 不重新抓取、不重新聚类、不调用 MiniMax，也不改变归档和音频逻辑。

## 页面结构

1. 页面顶部展示四个摘要指标：本期条目、决策收录、待补证、未收录。
2. 决策简报使用独立卡片逐条展示标题、中文概述、审核建议和证据评价；原播报稿折叠到“查看决策版播报稿”。
3. 情报全览按“已确认、热点关注、新兴信号”分组，每条独立换行展示标题、中文概述、关注理由和证据状态；原播报稿折叠到“查看全览版播报稿”。
4. 现有完整审核明细、证据链接、草稿编辑、排序、归档和音频操作继续保留。

## 数据与边界

- 查询层新增只读审核统计视图，不修改数据库结构。
- 未人工审核的条目不冒充“保留”，只依据 `included` 计入决策收录。
- 排除和重复分别保留准确计数，页面汇总为未收录，同时显示细分说明。
- 空分组显示清晰中文空状态。

## 验收

- 当前日报 1 应显示：本期 5、决策收录 3、待补证 1、未收录 2，并解释排除 1、重复 1。
- 决策版三条信息必须是三张独立卡片，而不是一整段文字。
- 情报全览五条应逐条显示并按类别分组。
- 桌面和窄屏均不出现横向溢出；长中文内容自然换行。
- 日报页面测试、完整 pytest 与 ruff 全部通过，并完成真实浏览器验收。
