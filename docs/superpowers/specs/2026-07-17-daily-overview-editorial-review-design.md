# 情报全览人工审核与可信语音设计

## 目标

在现有“决策简报 + 情报全览 + MiniMax 语音”能力上补齐报告级全览审核闭环。日报仍完整展示其事件操作快照中的全部可展示情报，但只有人工结论为 `keep`（保留）或 `needs_evidence`（需补证）的条目进入全览正文和全览语音；`exclude`（排除）、`duplicate`（重复）和未审核条目只保留在网页审计清单中。

本功能不重新抓取来源、不重新执行事件管线、不修改历史事件、RawItem 或来源状态，也不让 MiniMax 判断来源是否合法、事件是否可信或条目是否应收录。MiniMax 只负责把已确定的中文脚本转换为音频。

## 已有能力与缺口

现有 `daily_report_items` 是决策简报的精选快照，当前最多包含每个栏目限定数量的条目，并已有追加式人工审核记录。现有情报全览则由 `DailyReportQueryService._overview()` 直接读取日报绑定的完成事件操作快照，能够展示约 49 条 confirmed、hotspot 或 signal，但这些全览专属条目没有报告级审核记录。

因此当前存在以下缺口：

- 决策简报中的约 20 条可以人工保留、补证、排除或判重，全览剩余条目不能；
- 全览脚本直接使用全部可展示事件，弱证据、重复或不可信标题也会进入语音；
- 全览内容依赖查询时重建，缺少“这份日报当时审核了哪些候选项”的独立审计清单；
- 页面没有全览的审核完成度、排除原因和下一步动作汇总。

## 方案选择

采用独立的全览快照表和全览审核记录表。不能复用 `daily_report_items`，因为后者表达的是决策精选及其排序；把约 49 条全部塞入该表会破坏决策版语义、统计和现有页面。也不采用仅按证据根数自动过滤，因为规则无法可靠识别标题失真、事件重复和具有价值的单一权威信源。

## 数据模型

### `daily_report_overview_items`

每份日报保存一组不可变的全览候选快照：

- `id`：主键；
- `daily_report_id`：所属日报，日报删除时级联删除；
- `event_id`、`event_version_number`：来源事件及固定版本；
- `position`：全览稳定顺序，必须大于 0；
- `snapshot`：生成或修订日报时固定的中文标题、中文概述、影响说明、状态、展示层级、分类、发生时间、独立证据根数、确认摘要、限制说明和公开证据列表；
- `decision_item_id`：可空，指向同一日报中对应的决策简报条目，用于显示“已进入决策简报”，不用于改变决策条目的审核结论。

同一日报中的 `(event_id, event_version_number)` 和 `position` 分别唯一。快照不保存 Cookie、凭据、模型原始响应或外部网页正文，公开 URL 继续使用现有安全清洗规则。

### `daily_report_overview_editorial_reviews`

每次保存审核都追加一条不可变记录：

- `id`：主键；
- `daily_report_overview_item_id`：所属全览条目；
- `revision`：同一条目从 1 开始递增并联合唯一；
- `decision`：`keep`、`needs_evidence`、`exclude` 或 `duplicate`；
- `zh_title`：最终展示及播报使用的中文标题；
- `zh_summary`：中文文章概述；
- `review_recommendation`：中文审核建议和下一步动作；
- `evidence_assessment`：中文证据评价；
- `duplicate_of_overview_item_id`：仅 `duplicate` 可填写，必须指向同一日报中的另一条全览记录，且不能自引用；
- `created_at`：服务端 UTC 时间；
- `copied_from_editorial_review_id`：修订版复制来源，可空。

文本沿用现有决策审核字段的纯文本清洗和长度边界。`duplicate` 必须指定重复对象；其他结论不得指定重复对象。数据库负责基础约束，跨行归属和自引用由 repository 在事务中验证。

## 快照创建与历史兼容

新日报生成时，在同一事务内从已经确定的完成事件操作快照创建决策条目和全览条目。全览选择规则保持当前行为：纳入所有 `confirmed`，以及展示层级为 `hotspot` 或 `signal` 的事件；排序继续使用现有全览排序规则。单个事件快照无效时记录到生成摘要并跳过，不阻塞整份日报。

创建修订版时：

1. 父日报已有全览条目时，逐条复制其不可变快照；
2. 父日报是本功能上线前的历史日报、没有全览条目时，从父日报绑定的不可变完成事件操作快照一次性物化全览条目；
3. 父日报已有全览审核时，只复制每条记录的最新审核结论，并填写 `copied_from_editorial_review_id`；
4. 父日报没有全览审核时，新修订版中的条目保持未审核。

历史归档日报本身不回填、不修改。当前日报 4 将通过“创建修订版”得到可审核的新日报，避免改变已经归档并已有音频的历史成品。

## 审核与状态规则

只有草稿日报可以新增全览审核记录。保存单条审核时锁定日报和条目，验证条目归属及当前修订号，然后追加记录。单条失败只返回该表单的中文诊断，不影响其他条目。

全览的当前收录状态完全由最新审核结论派生：

- `keep`：进入全览正文和语音；
- `needs_evidence`：进入全览正文和语音，但必须显式播报“尚待进一步确认”及证据评价；
- `exclude`：不进入正文和语音，网页保留原因；
- `duplicate`：不进入正文和语音，网页显示被合并到哪一条；
- 未审核：不进入正文和语音，网页显示“尚未审核”。

决策简报审核和全览审核是两个独立视角。对应同一事件时，页面同时显示两边结论，但保存全览审核不会修改决策条目的 `included` 或历史审核记录。

## 正文、归档与语音一致性

全览正文构建器只接收 `keep` 和 `needs_evidence` 条目，并保持其稳定顺序。`needs_evidence` 的脚本段落必须包含风险提示和人工证据评价，不能只读经过润色的标题。

归档仍允许决策版先完成，不因全览未审核而阻塞现有决策日报流程。但是：

- 页面在存在未审核全览条目时显示“全览审核未完成”；
- `overview` 音频入队必须满足未审核数为 0 且至少有一个可播报条目，否则返回稳定中文错误；
- `decision` 音频规则保持不变；
- 草稿不可生成任何音频，归档日报不可继续编辑，因此同一日报内不存在“审核变化后旧音频仍有效”的竞态；
- 创建修订版会产生新的日报 ID 和独立音频，新版不会复用父日报音频。

Worker 继续在执行前校验日报已归档、脚本哈希与入队哈希一致。超时、有限重试、取消、心跳、恢复、结构化日志、密钥脱敏和中文诊断沿用现有实现。

## Web 页面与接口

日报详情页新增“情报全览审核”区域，页面顶部显示：

- 候选总数；
- 进入全览数；
- 需补证数；
- 排除数；
- 重复数；
- 未审核数。

全览列表默认显示全部候选项，并支持按审核状态和 confirmed/hotspot/signal 分组浏览。每条展示中文标题、中文文章概述、事件状态、来源证据、发布时间、证据根数、限制说明、是否进入决策简报、当前审核结论、中文审核建议和中文证据评价。长文本按段落换行，来源与审核信息使用清晰的视觉层级，不横向溢出。

草稿条目提供单条审核表单；归档页面只读展示审核历史。新增接口：

`POST /daily-reports/{report_id}/overview-items/{item_id}/editorial-reviews`

接口复用现有本机环回限制、同源验证、一次性动作令牌、模板转义和中文错误映射。路由只负责解析表单、调用 repository 并重定向，状态规则和归属验证不放在模板或路由中。

全览正文区域只渲染已收录条目；审核清单始终渲染全部条目。这样“不错过候选信息”和“不把不可信内容读进语音”能够同时成立。

## 组件边界

- `newsradar.db.models`：新增全览条目和追加式审核模型；
- Alembic 迁移：创建两张表、索引、唯一约束、检查约束和外键，支持 PostgreSQL 升降级；
- `newsradar.daily_reports.schema`：全览审核输入及验证；
- `newsradar.daily_reports.service`：新日报和历史修订版的全览快照物化；
- `newsradar.daily_reports.repository`：保存审核、归属/判重验证、复制最新审核；
- `newsradar.web.daily_report_queries`：读取全部全览候选、最新审核、历史和汇总，并构建可信正文脚本；
- `newsradar.web.app`：审核 POST 路由和中文错误映射；
- `daily_report_detail.html`：全览汇总、审计清单、表单、风险标签和可读性布局；
- `newsradar.operations` 与音频运行时：入队前检查全览审核完成度，现有 MiniMax 客户端和 Worker 机制不重写。

## 测试驱动与验收

实现时先写失败测试，再完成最小实现。测试覆盖：

- 数据模型和迁移的 PostgreSQL 升级/降级、唯一约束和检查约束；
- 新日报完整物化全览快照，单个坏事件不阻塞整份日报；
- 历史日报修订时从绑定操作快照物化，已有新式日报修订时复制快照和最新审核；
- 四种审核结论、文本边界、条目归属、重复对象归属、自引用、归档锁定和并发修订；
- 查询汇总、全部候选可见、正文只含 `keep`/`needs_evidence`、补证风险提示、重复关联和审核历史；
- Web 同源及动作令牌保护、HTML 转义、中文错误、长文本结构和筛选标签；
- 全览未审完、无可播报条目时拒绝音频入队，完成审核后脚本哈希和 Worker 成品一致；
- 决策版、归档、修订和既有音频测试不回归。

真实验收使用当前日报 4 创建修订版，完成全部约 49 条全览候选的逐项中文审核，确认网页统计与人工结论一致，再归档并生成一次真实 `speech-2.8-hd` 全览音频。最终执行完整 `pytest`、`ruff check .`、迁移测试和本地网页交互验收。未经用户确认不合并、不推送。
